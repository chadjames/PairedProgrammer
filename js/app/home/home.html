    WHAT TO DO AFTER SPINNING

  • Select spin above
  • Review kata requirements
  • Create a GitHub repository
  • Create the initial class and associated test class
  • Add the first broken test and push to your repository
  • Inform your chosen partner of the repository address
  • Wait for your partner to fix your broken test

    GENERAL GUIDELINES

  • DO NOT WRITE CODE WITHOUT YOUR PARTNER FIRST WRITING A BROKEN TEST
  • Tests should fail for a valid reason... i.e not for an exception
  • Use good commit messages, this is your way to communicate with your partner
  • Reactor when you see a good chance to improve the code without breaking existing tests
